Softwares de Versionamento

É um sistema que tem por objetivo realizar o controle de controle de versões de um código ou documento.

É possível gerenciar as versões de um código ou documento realizadas por um desenvolvedor ou equipe de desenvolvedores.

Software de Controle de Versões

Tem a finalidade de gerenciar diferentes versões de um documento. Diante disso, oferece uma maneira inteligente e organizada para facilitar o desenvolvimento de um projeto.

Está relacionado com as ferramentas de repositório local, histórico de alterações, repositório remoto, versão corrente, dentre outras. É uma forma eficiente de controle, tanto pessoal como externa, que envolve todo o caminho de desenvolvimento de arquivos.

Está subdividido em duas categorias:

Centralizado
Possui um repositório central em um servidor com controle de versões. É totalmente dependente do SERVIDOR. Um exemplo disso, são os Softwares Subversion.
Distribuido
Possui um repositório local com controle de versões. A atualização é feita através do Repositório Central. É independente do SERVIDOR. O exemplo disso, são os Softwares Mercurial.

GIT

É um software livre de versionamento que surgiu para o desenvolvimento do Kernel do Linux, foi projetado e desenvolvido por Linus Torvalds em 2005.

Cada diretório de trabalho do GIT é um repositório com um histórico completo e habilidade total de acompanhamento das revisões, não depende de acesso a uma rede ou a um servidor central.

GITHUB

É uma plataforma de hospedagem de código-fonte e arquivos com controle de versão.

Surgiu em 2008 e foi adquirido em 2018 pela Microsoft. Atualmente, hospeda em média, 100 milhões de projetos e possui mais de 56 milhões de usuários.

Comandos do GIT

  • Commit

    • Torna permanete um conjunto de alterações em um repositório.

  • Push

    • Envia alterações para o servidor no repositório remoto.

  • Fetch

    • Verifica se o seu repositor local está em sincronia com o remoto.

  • Pull

    • Sincroniza o repositório local com o remoto atualizando as versões.
